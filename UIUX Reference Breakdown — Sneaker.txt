UI/UX Reference Breakdown — Sneaker eCommerce Landing Page
1. Project Manager Perspective
Product Type

This is a fashion/sneaker eCommerce landing page designed to sell curated footwear collections with a strong lifestyle/editorial feel.

The page should feel like a mix of:

Premium sneaker store
Editorial fashion catalogue
Lifestyle lookbook
Direct-to-consumer product shop

The main objective is to move users from brand discovery → product interest → product browsing → purchase action.

Page Purpose

The page is built to:

Introduce the sneaker brand with a strong hero campaign.
Promote featured style categories.
Display product collections with pricing.
Recommend curated lifestyle collections.
Push seasonal collections.
End with a strong conversion banner and footer.
User Intent

The target user is likely browsing casually, not necessarily searching for one exact product.

The layout supports:

Visual discovery
Product comparison
Style inspiration
Quick buying decisions
Collection browsing
Content Priority

The page hierarchy is:

Hero campaign
Large emotional headline
Lifestyle sneaker image
Main CTA

Newly Dropped Collections

Recommended Collections
Lifestyle image grid
Summer Collections
Repeated product grid
Final CTA banner
Emotional brand statement
Footer
Navigation, newsletter, legal, social/payment-style links
Key User Actions

Primary actions:

Secondary actions:

Browse categories

3. Motion Graphics Developer Perspective
Recommended Animation Direction

The design should feel premium, smooth, and editorial — not overly playful.

Use:

GSAP for hero timeline, scroll reveals, image parallax, and section choreography.
Motion for lightweight hover/tap states, product cards, buttons, and small UI transitions.
Page Load Animation
Hero Timeline with GSAP

Sequence:

Header fades down.
Hero headline reveals line by line.
CTA fades up.
Model image slides in slightly from the right.
Carousel controls fade in.

Example direction:

const tl = gsap.timeline();

tl.from(".site-header", {
  y: -20,
  opacity: 0,
  duration: 0.6,
  ease: "power3.out",
})
.from(".hero-title-line", {
  y: 42,
  opacity: 0,
  stagger: 0.12,
  duration: 0.8,
  ease: "power4.out",
}, "-=0.2")
.from(".hero-cta", {
  y: 16,
  opacity: 0,
  duration: 0.45,
}, "-=0.3")
.from(".hero-image", {
  x: 80,
  opacity: 0,
  scale: 1.04,
  duration: 1,
  ease: "power4.out",
}, "-=0.7");
Scroll Reveal Animations

Use GSAP ScrollTrigger for:

Feature cards
Product cards
Recommended collection cards
Final CTA

Suggested behavior:

gsap.from(".product-card", {

RPC-First Backend Direction

Since you mentioned Supabase agent skills, business-critical operations should happen through RPCs.

Examples:

rpc_get_homepage_products()
rpc_get_collection_products(collection_slug text)
rpc_add_to_cart(user_id uuid, product_id uuid, quantity int)
rpc_create_checkout_session(cart_id uuid)
rpc_subscribe_newsletter(email text)

Frontend should not calculate:

Cart totals
Discounts
Availability
Stock limits
Tax/shipping
Final checkout price

The frontend should consume validated backend results.

Zustand Cart Store

Use Zustand only for client-side UI/cart draft state.

type CartState = {
  cartOpen: boolean;
  optimisticItems: CartItem[];
  openCart: () => void;
  closeCart: () => void;
};

Actual cart pricing and availability must come from Supabase RPCs.

Design requirements:
Create a centered max-width website layout on a light gray page background. The main content background should be warm off-white. The design should feel premium, editorial, earthy, warm, and fashion-focused.

Page sections:
1. Transparent overlay header with logo left, navigation center, utility icons right.
2. Large hero banner with warm brown background, left serif headline, CTA button, right model/sneaker lifestyle image, and bottom-right carousel controls.
3. Three promotional lifestyle cards under the hero, each with background image, dark overlay, text, and pill CTA.
4. "Newly Dropped Collections" product section with centered heading/subtitle and 3-column product grid.
5. Asymmetric recommended collections grid: one large left image card, two stacked right image cards, and one full-width banner card below.
6. "Summer Collections" product section using the same reusable product grid.
7. Final image CTA banner with large white serif text and centered CTA button.
8. Dark brown footer with logo, newsletter subscription, category links, legal links, help links, and subtle oversized brand mark.

Typography:
Use a heavy rounded serif display font for headings, similar to Fraunces/Recoleta/Cooper Black. Use Inter Tight or Geist for body, navigation, buttons, and product text.

Colors:
Use warm cream, beige, brown, dark chocolate, and off-white tones. Avoid harsh black except in small text. Avoid heavy shadows.

Product cards:
Each product card should have a soft off-white image container, centered sneaker image, price, small badge, product name, short description, Add to Cart button, and Buy Now button. Buttons should be small, rectangular with slight radius, with Add to Cart in light beige and Buy Now in brown.

Animations:
Use GSAP for the hero page-load sequence, scroll-triggered card/product reveals, parallax lifestyle images, and final CTA reveal. Use Motion for simple hover/tap interactions on product cards, buttons, and image cards. Keep all motion smooth, premium, and restrained.

Backend:
Use Supabase with an RPC-first architecture. Product data, collection data, pricing, stock, and checkout calculations must come from secure backend RPCs. The frontend should not calculate business-critical totals. Use TanStack Query for server data and Zustand only for client UI state such as cart drawer open/closed or temporary optimistic state.

Responsive:
Desktop should match the reference with 3-column product grids and asymmetric lifestyle layouts. Tablet should reduce to 2 columns. Mobile should stack sections cleanly, use a hamburger menu, and avoid squeezing desktop layouts.
